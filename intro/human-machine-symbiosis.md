# 人机协作治理数字资产

这篇工作札记解读已发布的《量潮数字资产标准 v0.0.3》和《量潮数字资产云工作愿景（2026 年 4 月版）》的建模思想。

## 为什么需要数字资产治理？


从 2016 年到 2026 年，我们的数字资产已经积累了整整十年。随着 AI 原生时代的到来，我们的数字资产迎来了空间的爆炸性增长。


这导致我们的数字资产面临着数量多、增长快、时间线长、信息密度大的问题。我们希望打破当前只能靠少数核心管理和传递资产的手工业模式，让智能体能够参与到数字资产治理中。

## 数字资产平台的定位是什么？


数字资产平台提供一个中央仓库。在资产进入公司空间、内部交付和客户交付等场景下，我们需要检查资产是否齐全、是否符合格式规范、是否符合维护流程等。


我们希望通过它能够看清整个组织内部纷繁芜杂的数字资产的全貌，为治理数字资产及其生产和维护流程提供决策支持。我们希望通过它给日益爆发的智能体的生产力准备一根缰绳。


## 如何对数字资产治理问题建模？


采取“契约驱动治理”的建模思想。在整个体系中，围绕代表治理意图的“数字资产契约”和实际现状的“数字资产目录”，围绕发现、注册、验证等治理行为操作。


最终期望人类意图与契约一致、契约与目录一致。智能体可以帮助人类有效维护并应对变化动态更新，人类可以从枯燥繁重的治理工作中解放，专注于发挥自己的创造力。


## 落实到哪些具体场景？


我们有大量的标准数字资产用来支撑我们的法治实践。其中 GitHub 有一组开源仓库，即前文介绍过的量潮第二大脑系统。同时飞书还有一套日常协作的知识库用于生产。


我们需要平台帮助我们集中同类资产的信息，以帮助我们知道如何整合他们。基本原则是飞书知识库提炼精华公开到 GitHub 仓库，再精选当前有业务意义的资产发布到公众号等新媒体。


## 如何组织平台研发？


我们首先会围绕一部分案例写示例代码，通过编码过程让隐性治理流程显性化，通过不断重构成符合标准文件的建模方式。平台的全貌会在反复重构和验证新案例的过程中自然地被人类和 AI 协作“蒸馏”出来。


在平台开始接管一部分工作以后，我们开始把一部分精力逐渐转向观察系统和迭代新功能。随着这个过程的加速，平台的工作范围会很快进入指数级爆炸，直到覆盖我们的大部分场景。


当平台稳定以后，我们将会发布 V1.0.0 并开启商业化进程和融资进程。
